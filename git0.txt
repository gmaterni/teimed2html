Questi sono i comandi Git comuni usati in varie situazioni:

crea un'area di lavoro (vedi anche: git help tutorial)
   clone             Clona un repository in una nuova directory
   init              Crea un repository Git vuoto o ne reinizializza uno esistente

lavora sulle modifiche correnti (vedi anche: git help everyday)
   add               Aggiunge i contenuti del file all'indice
   mv                Sposta o rinomina un file, una directory o un collegamento simbolico
   restore           Ripristina i file nell'albero di lavoro
   rm                Rimuove file dall'albero di lavoro e dall'indice
   sparse-checkout   Inizializza e modifica sparse-checkout

esamina la cronologia e lo stato (vedi anche: git help revisions)
   bisect            Usa la ricerca binaria per trovare il commit che ha introdotto un bug
   diff              Visualizza le modifiche fra commit, un commit e l'albero di lavoro, ecc.
   grep              Stampa le righe corrispondenti a un'espressione regolare
   log               Visualizza i log dei commit
   show              Visualizza vari tipi di oggetti
   status            Visualizza lo stato dell'albero di lavoro

aumenta, contrassegna e metti a punto la cronologia comune
   branch            Elenca, crea o elimina branch
   commit            Registra modifiche al repository
   merge             Unisce due o pi√π cronologie di sviluppo
   rebase            Riapplica dei commit dopo l'ultimo commit di un branch
   reset             Ripristina l'HEAD corrente allo stato specificato
   switch            Passa da un branch a un altro
   tag               Crea, elenca, elimina o verifica un oggetto tag firmato con GPG

collabora (vedi anche: git help workflows)
   fetch             Scarica oggetti e riferimenti da un altro repository
   pull              Esegue il fetch e l'integrazione con un altro repository o un branch locale
   push              Aggiorna i riferimenti remoti insieme agli oggetti associati

'git help -a' e 'git help -g' elencano i sottocomandi disponibili e
alcune guide concettuali. Vedi 'git help <comando>' o 'git help <concetto>'
per leggere ulteriori informazioni su un sottocomando o su un concetto
specifici.
Vedi 'git help git' per una panoramica del sistema.
